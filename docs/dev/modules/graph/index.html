concept EdgeDirectionLike -- definiujący kierunek krawędzi, wraz z potencjalną wagą.

concept EdgeLike -- concept definiujący krawędź, z wyróżnionym początkiem(source) i końcem(target) i potencjalną wagą.
Zauważmy, że każdy EdgeLike w szczególności spełnia EdgeDirectionLike.
DirectableEdge -- concept definiujący możliwość krawędzi do porzucenia punktu startowego.

getDirection(Edge) -- funkcja pozwalająca wyciągnąć kierunek z danej krawędzi. Zauważmy, że działa niezależnie od spełnienia DirectableEdge.

EdgeDirection -- domyślna implementacja interface kieruneku krawędzi, pozwala przechowywać wagę
Edge -- domyślna implementacja interface krawędzi, pozwala przechowywać wagę
GraphReprezentation -- definiuje przechowywanie grafu w celu wyciągnięcia ciężko dostępnych informacji jak listę sąsiedztwa, lub istnienie krawędzi. Jest to narzędzie niezbędne w większości algorytmów. wymagania GraphReprezentation:
  • void addEdge(EdgeLike)
  • void addVertex()
  • range getIncidentEdges(Vertex)
Jako funkcje pomocnicze implementuje:
  • void addEdges(range)
  • void addVertecies(count)
class Graph -- jest reprezentowany przez podane GraphReprezentation oraz kolejność krawędzi. Jako, że krawędź w zależności od skierowania jest incydentalna z jednym lub dwoma wierzchołkami to graph w ogólnym pojęciu nie jest w stanie zdefiniować funkcji dodania krawędzi. class UndirectedGraph -- class DirectedGraph --
